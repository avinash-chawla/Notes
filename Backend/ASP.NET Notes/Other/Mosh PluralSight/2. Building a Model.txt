1. Create new Project

--> Select .net framework, version 4.5, mvc template, individual authentication
--> Name it: Gighub

--> enable-migrations
--> add-migration "InitialModel"
--> update-database

--> When you choose mvc template, a default connection string is established in web.config refering SqlServerExpress Edition
--> You can see this DB in SQL Server Explorer.
--> You can also see mdf file in App_Data folder, by clicking on 'show all files'

---------------------------------------------------------------------------------

2. Creating Domain Class

--> Make new class in Models folder --> Gig.cs

public class Gig
    {
        public int Id { get; set; }
        public ApplicationUser Artist { get; set; }
        public DateTime DateTime { get; set; }
        public string Venue { get; set; }
        public Genre Genre { get; set; }
    }

--> There is inbuilt domain class --> 1. ApplicationDbContext and 2. ApplicationUser
--> Both classes are available in IdentityModel.cs file
--> Seperate both classes in different and keep name same as class Names.

--> Nuget Package: Productivity Power Tools is now part of asp.net
--> Tools --> options --> Productivity Power tools --> Power Command --> General --> Tick Remove and Sort usings on Save

--> In ApplicationDbContext class,

        public DbSet<Gig> Gigs { get; set; }
        public DbSet<Genre> Genres { get; set; }

--> add-migration CreateGigDomain
--> update-database

--> Above properties of domain class will automatically create foreign key to Genre and ApplicationUser table with their Ids.
